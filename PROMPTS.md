# 🧠 ClimateVault: Prompt Engineering & Architecture Library

This document outlines the detailed prompt engineering process used to architect, build, and refine the **ClimateVault** platform. It serves as a blueprint for the logic, API integrations, data orchestration, and UI/UX design decisions made during the build.

---

## 🏗️ Phase 1: The Foundation & UI Design
**Objective:** Establish a high-end, dark-themed FinTech aesthetic and the core asset-input framework.

> "Build a React web app called 'ClimateVault' with the tagline 'Know Your Asset's Future Before the Market Does.' Use a premium dark theme: background #0A0F1E (very dark navy), cards #131B2E, accent color #00D4FF (electric cyan). The layout should be: A top navigation bar with a minimalist logo (White square vault outline with a glowing Cyan 'C' inside). Below that, a two-column layout: LEFT COLUMN (40%) for Asset Details inputs (Value, Type, Year, Country - full 195 list) and RIGHT COLUMN (60%) for an Interactive Risk Map placeholder. Make it fully mobile responsive with flex-wrapping."

---

## 🗺️ Phase 2: Geospatial Intelligence
**Objective:** Integrate real-world mapping and geocoding logic.

> "Replace the map placeholder with a real interactive map using 'mapbox-gl' and 'react-map-gl'. Use Mapbox credentials for the TileLayer: `mapbox://styles/mapbox/dark-v11`. Default center to Dubai. When the user types an address and clicks analyze, use OpenStreetMap Nominatim API to find and fly to that location. Drop a glowing cyan pin and draw three concentric hazard circles: Red (500m Extreme Risk), Orange (1500m High Risk), and Yellow (3000m Moderate Risk)."

---

## 📈 Phase 3: Financial Risk Modeling
**Objective:** Translate environmental hazard data into compounding financial depreciation curves.

> "Add a 'Financial Risk Projection' section with a data table showing 10-year intervals extending from 2025 to 2085. Columns: Year, Climate Scenario (RCP 4.5/8.5), Projected Asset Value, Value Loss ($), Value Loss (%), and Risk Level. Implement a Recharts line chart showing 'Asset Value Depreciation Curve' with two lines: 'Business as Usual' (declining) vs. 'With Adaptation Investment' (stable). Dynamically calculate based on user-inputted asset value."

---

## 📡 Phase 4: The "Brain" (Multi-API Integration & Fallbacks)
**Objective:** Synthesize real-time data from 6 global environmental databases with a resilient "Fail-Safe" pipeline.

> "Architect an `apiService.ts` to handle live data fetching. CRITICAL RULE: If ANY API fails or times out, it must FAIL SILENTLY and return realistic regional fallback data so the UI NEVER crashes. 
> 1. **Elevation:** Fetch from Open-Meteo. 
> 2. **Earthquake:** Fetch from USGS Earthquake Catalog. Count Mag 4.0+ in 200km over 25 years. 
> 3. **Climate:** Fetch temp, precipitation, and windspeed from Open-Meteo. 
> 4. **Coastal/River:** Fetch from Overpass API. *Optimization:* Use `out center;` and 5km radii to prevent 504 Gateway Timeouts.
> 5. **Dynamic UI:** Render Risk Factor cards showing '🟢 LIVE DATA' or '🟡 ESTIMATED' badges based on API success."

---

## ⚖️ Phase 5: CSRD Compliance & Double Materiality
**Objective:** Align the platform with 2026 European Sustainability Reporting Standards (ESRS).

> "Implement **Double Materiality** logic. Update inputs to accept 'Annual Energy Usage' and 'Fuel Source'. In `apiService.ts`, calculate a Carbon Footprint Impact Score. Build a `MaterialityMatrix.tsx` component that draws a 2x2 grid plotting Financial Materiality (Physical Risk to Asset) on the X-axis against Impact Materiality (Carbon Risk to Planet) on the Y-axis. Plot a glowing cyan dot at the exact intersection."

---

## 🚀 Phase 6: The "Billion Dollar" Metrics (Differentiation)
**Objective:** Solve complex institutional problems (Obsolescence, Regulatory Risk, Neighborhood Failure).

> "Upgrade `apiService.ts` to calculate:
> 1. **Stranded Asset Year:** Predict the exact year of obsolescence based on compounding risk tipping points. Display in a massive 'Countdown Clock' banner.
> 2. **Infrastructure Severance (Island Effect):** Use Overpass API to scan for critical infrastructure (hospitals, grids) within 3km to calculate probability of neighborhood isolation.
> 3. **Transition Risk:** Calculate projected annual carbon tax penalties (e.g., Local Law 97) for older commercial assets.
> 4. **Adaptation ROI & Green Premium:** Calculate market value uplift generated by executing climate resilience upgrades."

---

## 📊 Phase 7: Enterprise Bulk Upload & Rate Limiting
**Objective:** Allow institutional users to analyze entire property portfolios simultaneously.

> "Build a `BulkUploadZone.tsx` using `papaparse` to accept CSV portfolio uploads. In `Index.tsx`, implement a sequential asynchronous queue that processes properties one-by-one. **CRITICAL:** Add a 2500ms (2.5 second) delay between API calls to prevent IP bans from free geocoding rate-limits. Generate a Master Export CSV containing all 19 data columns for the entire portfolio."

---

## 📄 Phase 8: Institutional PDF Reporting Engine
**Objective:** Generate audit-ready executive disclosures natively.

> "Uninstall canvas-based screenshot tools. Build a custom 7-page reporting engine inside `ExportButton.tsx` using `jsPDF`. Implement a dynamic `checkPageBreak` layout engine to prevent text overlapping. The report must include a dark-mode cover page, Double Materiality breakdown, Hazard Profiles with API source citations, and an Adaptation Strategy roadmap."

---

## 🔐 Phase 9: SaaS Authentication & Security
**Objective:** Transition from a stateless tool to a secure SaaS product.

> "Integrate `@supabase/supabase-js`. Create an `AuthModal.tsx` with a glassmorphism UI for Sign In/Sign Up. Connect to `Navbar.tsx` using `useEffect` to listen to auth state changes. Manage 'Z-Index Wars' by assigning `z-[9999]` to the modal so it properly overlays the greedy Mapbox canvas. Add official Copyright and Licensing headers to core logic files."

* **APIs:** Nominatim (OSM), Open-Meteo, USGS, Overpass API
* **Backend/Auth:** Supabase
* **CI/CD Deployment:** Vercel (Primary), Netlify (Fail-safe Backup)
